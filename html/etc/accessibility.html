웹 콘텐츠 접근성 지침(WCAG, Web Content Accessibility Guidelines)

접근성 지침은 웹 콘텐츠가 모든 사용자, 특히 장애를 가진 사용자들에게 쉽게 접근될 수 있도록 하는 원칙들을 제시.

1. 인식 가능(Perceivable)

사용자가 콘텐츠를 인식할 수 있어야 함.
정보와 사용자 인터페이스 구성 요소를 시각적, 청각적으로 인식할 수 있어야 함.

대체 텍스트 제공

이미지, 비디오 등 시각 콘텐츠에 텍스트 대체 설명을 제공하여 시각 장애인도 이해할 수 있도록 해야 함.

자막 제공

동영상 콘텐츠에는 자막을 추가해 청각 장애인이 내용을 이해할 수 있게 함.

명확한 레이아웃

글자 크기, 색상 대비를 충분히 높여 텍스트 가독성을 높여야 함.

2. 운용 가능(Operable)

사용자가 인터페이스를 쉽게 조작할 수 있어야 함.
특히, 마우스를 사용할 수 없는 사용자들을 고려한 기능이 중요.

키보드 접근성

모든 기능은 키보드만으로도 조작 가능해야 함. 마우스를 사용할 수 없는 사용자를 배려해야 함.

충분한 시간 제공

시간 제한이 있는 콘텐츠에서는 사용자가 충분한 시간을 가지고 상호작용할 수 있게 설정해야 함.

발작 예방

깜박임이나 빠른 애니메이션 효과는 발작을 유발할 수 있으니 주의

3. 이해 가능(Understandable)

사용자가 콘텐츠와 인터페이스를 쉽게 이해할 수 있어야 함.

일관된 네비게이션

사용자는 각 페이지에서 일관된 방법으로 이동할 수 있어야 함. 메뉴나 버튼의 위치가 일관되게 유지되는 것이 중요

명확한 언어 사용

콘텐츠는 사용자가 이해할 수 있는 수준의 언어로 작성해야 하고, 이해하기 어려운 용어에는 설명을 제공

입력 오류 방지

폼에서 입력 오류가 발생했을 때, 사용자에게 오류를 설명하고 해결 방법을 제공

4. 견고성(Robust)

콘텐츠는 다양한 사용자 에이전트(예: 스크린 리더)와 호환 가능해야 하고, 최신 기술과도 잘 연동되어야 함.

웹 표준 준수

HTML, CSS 등 표준을 준수하여 콘텐츠가 다양한 브라우저, 보조 기술과 호환되도록 해야 함.

정확한 코드 작성

구문 오류가 없는 HTML, CSS, JavaScript 코드를 작성해서 보조 기술이 올바르게 콘텐츠를 해석할 수 있게 해야 함.

추가적으로 알아두면 좋은 점

  • - 콘텐츠는 쉽게 확대/축소가 가능해야 하고, 최소한 200% 확대해도 레이아웃이 깨지지 않도록 해야 함.
  • - 자동으로 재생되는 음악이나 비디오는 사용자가 제어할 수 있도록 재생 중지 버튼을 제공해야 함.

구체적인 예

1. 대체 텍스트(Alt Text) 사용

예시: 웹 페이지에 이미지를 삽입할 때,
시각 장애 사용자가 스크린 리더로 이미지를 인식할 수 있도록 <img> 태그에 alt 속성을 설정해야 함.

<img src="profile.jpg" alt="달리는 여성의 프로필 사진">

·나쁜 예: alt="이미지" - 이미지 내용을 설명하지 않음.
·좋은 예: alt="노을을 배경으로 달리고 있는 여성의 모습" - 이미지를 명확하게 설명함.

2. 자막 제공

예시: 비디오 콘텐츠가 있을 때, 자막을 제공해 청각 장애인도 내용을 이해할 수 있도록 해야 함.
자막은 HTML5 <track> 태그를 사용해 추가 가능.

<video controls>
  <source src="movie.mp4" type="video/mp4">
  <track kind="subtitles" srclang="en" src="subtitles_en.vtt" label="English">
</video>

·나쁜 예: 자막 없이 비디오만 제공.
·좋은 예: 자막 파일을 추가하여 청각 장애 사용자도 비디오 내용을 이해할 수 있게 제공.

3. 키보드로 접근 가능한 메뉴

예시: 키보드만으로도 웹 페이지의 모든 메뉴와 버튼을 탐색할 수 있도록 만들어야 함.
특히, tabindex 속성을 사용해 키보드 네비게이션을 관리할 수 있음.

<nav>
  <ul>
    <li><a href="#home" tabindex="0"></a></li>
    <li><a href="#about" tabindex="0"></a></li>
    <li><a href="#contact" tabindex="0"></a></li>
  </ul>
</nav>

·나쁜 예: 메뉴가 마우스로만 클릭할 수 있는 상태.
·좋은 예: tab 키로 메뉴 항목을 선택하고, enter로 활성화 가능.

4. 충분한 텍스트 대비 제공

예시: 배경색과 텍스트 색상의 대비를 높여 시각적 인식이 어려운 사용자도 텍스트를 쉽게 읽을 수 있도록 해야 함.
일반적으로 텍스트와 배경 간 대비 비율은 4.5:1 이상이어야 함.

body {
  background-color: #ffffff; /* 하얀 배경 */
  color: #000000; /* 검은 텍스트 */
}

·나쁜 예: 흐린 회색(#cccccc) 텍스트와 하얀 배경을 사용하여 가독성이 떨어짐.
·좋은 예: 검은색 텍스트(#000000)와 하얀 배경(#ffffff)으로 높은 대비를 제공.

5. 명확한 폼 필드 레이블 제공

예시: 사용자 입력 폼에서 각 입력 필드에 명확한 레이블을 연결해 시각 장애 사용자들이 스크린 리더로 인식할 수 있게 해야 함.
label 태그를 사용해서 폼 필드와 연관시키면 좋음.

<label for="name">이름:</label>
<input type="text" id="name" name="name>

·나쁜 예: 폼 필드에 레이블이 없거나 placeholder만 사용.
·좋은 예: 명확한 레이블을 사용하고 for 속성으로 입력 필드와 연결.

6. 콘텐츠를 자동으로 갱신하지 않기

예시: 자동으로 변하는 배너나 팝업은 사용자에게 혼란을 줄 수 있음.
대신, 갱신하거나 변할 때는 명확하게 사용자 제어가 가능하도록 설정해야 함.

<div role="alert" aria-live="polite">새로운 알림이 있습니다.</div>

·나쁜 예: 자동으로 팝업이 뜨고 사라지는 광고.
·좋은 예: 사용자가 직접 닫거나 무시할 수 있는 알림 제공.

7. 페이지 구조를 시맨틱하게 작성

예시: 시맨틱 HTML 요소(<header>, <main>, <nav>, <footer> 등)를 사용해 스크린 리더가 페이지 구조를 쉽게 이해하도록 작성.

<header>
  <h1>웹 사이트 제목</h1>
</header>
<nav>
  <ul>
    <li><a href="#home">Home</a></li>
    <li><a href="#services">Services</a></li>
  </ul>
</nav>
<main>
  <article>
    <h2>글 제목</h2>
    <p>본문 내용</p>
  </article>
</main>
<footer>
  <p>저작권 정보</p>
</footer>

·나쁜 예: <div>만으로 모든 구조를 작성.
·좋은 예: 적절한 시맨틱 태그를 사용해 콘텐츠를 구조화.

8. 애니메이션의 사용자 제어 기능 제공

예시: 페이지에서 자동으로 움직이는 애니메이션이 있다면 사용자가 멈출 수 있도록 기능을 제공해야 함.

<div class="slider">
  <button aria-label="슬라이더 멈추기">정지</button>
</div>

·나쁜 예: 자동으로 계속 재생되는 애니메이션.
·좋은 예: 정지 버튼을 제공하여 사용자가 직접 제어할 수 있게 함.

접근성 체크리스트

1. 대체 텍스트(Alt Text)

이미지에 적절한 대체 텍스트가 있는가?
모든 이미지에는 시각적으로 제공된 정보를 설명하는 대체 텍스트가 있어야 함.

2. 키보드 접근성

모든 기능을 키보드만으로 사용할 수 있는가?
사용자 인터페이스는 키보드로 네비게이션이 가능해야 하고, 탭 순서가 논리적이어야 함.

3. 색상 대비

텍스트와 배경 간의 색상 대비가 충분한가?
텍스트와 배경의 대비 비율이 최소 4.5:1 이상인지 확인해야 함.

4. 자막과 대체 콘텐츠

비디오와 오디오 콘텐츠에 자막이나 대체 텍스트가 제공되는가?
청각 장애인을 위해 동영상에는 자막, 오디오 콘텐츠에는 대체 텍스트를 제공해야 함.

5. 명확한 폼 레이블

폼 필드에 명확한 레이블이 있는가?
입력 필드에는 화면 리더기 사용자를 위한 레이블이 있어야 함.

6. 링크 텍스트

링크 텍스트가 명확한가?
"여기를 클릭"이 아니라, 링크 자체가 명확한 목적을 설명해야 함. 예: "다운로드 페이지로 이동".

7. 페이지 구조

시맨틱 HTML 태그를 사용했는가?
<header>, <nav>, <main>, <footer>와 같은 시맨틱 태그를 사용해 구조가 명확해야 함.

8. 콘텐츠의 시간 제어

자동 재생 콘텐츠를 멈출 수 있는 기능이 있는가?
슬라이드쇼, 비디오 등 자동으로 변하는 콘텐츠는 사용자가 제어할 수 있어야 함.
