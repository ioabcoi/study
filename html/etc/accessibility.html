접근성 지침

1. 인식의 용이성


1) 대체텍스트

① alt

  • ⑴ 링크로 묶여있는 경우 빈 값으로 제공

    <a><img alt="" /><span>제목</span></a>

  • ⑵ 링크 없이 제공되는 경우 제목과 동일한 대체텍스트 제공

    <img alt="제목" /><span>제목</span>

② longdesc

  • alt만으로 설명이 불가능할 경우 숨김 텍스트 제공
    • <img longdesc="#explain" /><div id="#explain" class="hidden">내용</div>
    • (단, 숨김 텍스트도 구조적으로 제공해야 함)

2) 자막 제공

  • 동영상, 음성 콘텐츠는 자막, 대본/원고, 수화 제공

    사용자의 필요에 따라 자막을 끄고 켤 수 있도록 제공, 대사가 없는 경우엔 화면 해설을 제공


3) 명료성

  • ① 명도대비

    18pt 이상 또는 14pt 이상 굵은 폰트의 경우 3:1

  • ② 색에 무관한 콘텐츠 인식

    GNB/ LNB/ paging 등 선택된 항목들에게 색의 변화가 아닌 다른 방법으로 구분 가능하도록 제공 (ex: 밑줄, 불릿 등)

  • ③ 명확한 지시사항

    색, 크기, 모양 또는 위치에 관한 정보를 인식하지 못하더라도 원하는 콘텐츠에 접근할 수 있도록 제작

    ex) 동그란 버튼/ 오른쪽 버튼을 누르시오 (X)

    ex) 실제 명칭을 제공하거나 그 요소의 대체텍스트를 사용해 지칭 : 더보기 버튼/ 지도보기 버튼을 누르시오 (O)

  • ④ 배경음 사용 금지

    자동으로 재생되는 배경음악 사용 금지

    (단, 3초 미만은 예외 / 3초 이상일 경우 마우스 또는 키보드로 제어할 수 있는 수단 제공)

2. 운용의 용이성


1) 키보드 접근성

  • 마우스 이벤트와 키보드 이벤트 병행

  • ① 모든 기능은 키보드로 사용할 수 있어야 함

    스크롤이 생기는 컨텐츠 블록일 경우 tabindex="0"으로 초점을 받아 스크롤을 사용할 수 있도록 해야 함

  • ② 초점은 논리적으로 이동해야며 시각적으로 구별할 수 있어야 함
    • ⑴ onfocus="this.blur();" outline:none; 사용 금지
    • ⑵ cursor="pointer" 로 스타일 지정
    • ⑶ flash wmode 값 transparent 또는 opaque
  • ③ 이미지 슬라이드

    마크업 / 초점 순서 - ◀□□□□(□□□□)▶ / 마지막 페이지로 넘어갔을 때 ▶버튼에 '마지막페이지'임을 명시


2) 충분한 시간 제공

  • ① 응답시간 조절

    시간제한 있는 콘텐츠는 응답시간 조절할 수 있어야 함

  • ② 정지기능 제공

    자동으로 변경되는 콘텐츠는 움직임을 제어할 수 있도록 정지기능 제공 - 정지, 이전, 다음


3) 쉬운 네비게이션

  • ① 스킵 네비게이션

    반복영역 건너뛰기 (본문 바로가기)

    • ⑴ 페이지 가장 처음에 위치
    • ⑵ 배경음이 있을 경우 배경음 다음에 위치
  • ② 제목 제공

    페이지, iframe, 컨텐츠 블록에는 적절한 제목을 제공

    • ⑴ 페이지 타이틀 - 팝업창일 경우 '팝업창' 명시
    • ⑵ iframe - title="제목" / 내용이 없는 경우 title="빈 프레임" src="about:blank" 제공
    • ⑶ 컨텐츠 블록 - 제목을 제공하는 경우 헤딩태그 부여 / 컨텐츠 블록이 없는 경우 헤딩태그 사용 안됨
  • ③ 적절한 링크 텍스트

    숨김 텍스트로 더 구체적으로 명확하게 제공

    • ⑴ '더보기' 링크의 경우 - '공지사항 더보기', '이벤트 더보기'
    • ⑵ 한 페이지에 동일한 링크 텍스트가 있을 경우 - '미디어포스원 지도보기', '이수역 지도보기'
    • ⑶ 이미지 링크일 경우 - '홈페이지로 이동', '첫번째 페이지로 이동', '다음 페이지로 이동'
    • ⑷ 활성화를 시각적으로만 확인할 수 있을 때는 숨김텍스트로 '활성화' 명시

3. 이해의 용이성


1) 기본언어 표시

  • <html lang="ko"><html xmlns="http://www.w3.org/1999/xhtml" lang="ko" xml:lang="ko">


2) 사용자 요구에 따른 실행

  • 마우스 이벤트와 키보드 이벤트 병행

  • ① 마우스 오버
  • ② 마우스 클릭
  • ③ 드롭다운 메뉴

    선택과 조작의 단계를 통해 구현되어야 함 (셀렉트박스에서 옵션 선택 후 이동 버튼을 클릭해야 페이지가 이동)

  • ④ 풀다운 메뉴
  • ⑤ 새 창
    • ⑴ target="_blank"
    • ⑵ target="_blank" 없이 스크립트를 사용할 경우 - title="새 창"
    • ⑶ alt에 함께 제공 - alt="인쇄하기 새 창"

3) 컨텐츠 선형화

  • ① 콘텐츠의 논리적인 초점 순서

    좌 -> 우, 상 -> 하

  • ② 탭, 컨텐츠
    • ⑴ 한 페이지에 모든 탭과 컨텐츠가 있는 경우 - 탭1, 컨텐츠1, 탭2, 컨텐츠2, 탭3, 컨텐츠3…
    • ⑵ 각각 다른 페이지에 컨텐츠가 있는 경우 - 탭1, 탭2, 탭3… , 컨텐츠n (현재 활성화된 탭n의 제목을 컨텐츠의 제목으로 명시해주어야 함)
  • ③ 이미지 슬라이드

    이전버튼 -> 이미지1, 이미지2, 이미지3, .... -> 다음버튼

  • ④ 레이어 팝업

    열기 버튼 -> 레이어 팝업 -> 닫기


4) 표의 구성

  • ① caption

    표의 내용을 요약한 정보를 제공

  • ② th

    scope 제공


5) 레이블 제공

  • <input>, <textarea>, <seelect> 등의 입력 서식에는 각각 대응하는 레이블을 제공해야 함

    휴대폰번호, 이메일주소 처럼 하나의 레이블로 대응이 안될 경우 title="" 제공

4. 견고성


1) 문법 준수

  • 마크업 오류 방지
    • - 태그의 열고닫음, 중첩, 중복된 선언, id 속성 값
    • - Markup Validation Service - http://validator.w3.org/
    • - 자동검사 k-wah / 수동검사 openWAX, n-WAX

접근성 체크리스트

1. 인식의 용이성


1) 대체텍스트

  • - <img> <input type="image"> <area> 의 alt 속성이 있는지, 적절한 대체텍스트를 제공하였는지 체크
  • - img와 텍스트가 링크로 묶여있는 경우, img alt=""는 빈값으로 제공하였는지 체크 (중복 제공되었는지 체크)
  • - 긴 내용의 숨김 텍스트도 (h1h2h3..., p, ul 등을 사용하여) 구조적으로 제공되었는지 체크

2) 자막 제공

  • - 동영상에 자막, 원고 또는 수화가 제공되었는지 체크

3) 명료성

  • - GNB/ LNB/ paging 등 선택된 항목들에게 색의 변화가 아닌 다른 방법으로 구분 가능한지 체크
  • - 3초 이상의 배경음악이 자동으로 재생되는지 체크 (자동으로 재생된다면 마우스 또는 키보드로 제어할 수 있도록 제공되었는지 체크)

2. 운용의 용이성


1) 키보드 접근성

  • - Tab 과 Shift + Tab 으로 모든 컨텐츠에 접근이 가능하도록 제공되었는지 체크
  • - 컨텐츠 접근 순서가 논리적인지 체크
  • - 초점 받은 요소가 시각적으로 구분 가능한지 체크
  • - flash wmode 값이 transparent 또는 opaque 인지 체크
  • - onfocus="this.blur();" outline:none; 속성/ 스타일을 사용하였는지 체크

2) 충분한 시간 제공

  • - 시간제한 있는 콘텐츠는 응답시간 조절할 수 있도록 제공되었는지 체크
  • - 자동으로 변경되는 콘텐츠는 움직임을 제어할 수 있도록 정지기능이 제공되었는지 체크

3) 쉬운 네비게이션

  • - 반복영역 건너뛰기(본문 바로가기)가 제공되고 포커스가 제대로 이동되는지 체크
  • - 키보드로도 제대로 작동이 되는지 체크
  • - 페이지, iframe 에 적절한 title이 제공되었는지 체크
  • - 컨텐츠 블록에 헤딩태그가 적절하게 제공되었는지 체크

3. 이해의 용이성


1) 기본언어 표시

  • - <html> 요소 lang 속성 값 체크

2) 사용자 요구에 따른 실행

  • - 새 창으로 동작하는 a태그에 target="_blank"를 제공되었는지 체크

3) 컨텐츠 선형화

  • - 컨텐츠의 순서가 논리적으로 제공되었는지 체크
  • - 탭의 경우, 탭과 컨텐츠의 순서가 논리적으로 제공되었는지 체크
  • - 레이어 팝업의 경우, 레이어 팝업의 위치가 논리적으로 제공되었는지 체크

4) 표의 구성

  • - caption 내용이 올바르게 제공되었는지 체크
  • - th 요소가 있는지, scope 속성이 적절히 제공되었는지 체크

5) 레이블 제공

  • - <input>, <textarea>, <seelect> 요소 등에 <label> 과 title 이 올바르게 제공되었는지 체크

4. 견고성


1) 문법 준수

  • - 태그의 열고 닫음 체크
  • - 태그의 중첩이 올바른지 체크
  • - 한 요소에 중복된 속성 선언 여부 체크
  • - 한 페이지에 중복된 id 속성 값 선언 여부 체크
  • - Markup Validation Service - http://validator.w3.org/
  • - 자동검사 k-wah / 수동검사 openWAX, n-WAX
